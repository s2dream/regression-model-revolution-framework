# [AutoML Regression Framework] 역요구사항 명세서 (Reverse Requirements Specification)

본 문서는 현재 구현되어 있는 **AutoML Regression Framework**의 소스 코드와 `ARCHITECTURE.md` 파일을 리버스 엔지니어링하여 시스템의 설계 의도, 핵심 기능, 그리고 품질 속성을 역으로 도출한 **요구사항 명세서**입니다.

---

## 1. 개요 (Overview)

### 1.1 시스템 정의
본 시스템은 정형 데이터(Tabular Data)를 입력받아 다양한 기계학습 회귀(Regression) 알고리즘을 활용해 성능을 일괄 학습하고 예측하는 **AutoML Regression Framework**입니다. 

### 1.2 시스템 목표
- **종합 솔루션 제공**: 데이터 수집, 전처리, 모델 학습, 성능 평가, 고화질 분석 플롯 생성 및 결과 보고서 저장을 아우르는 단일 파이프라인을 구축합니다.
- **개발 부담 최소화**: 기계학습 모델 훈련에 소요되는 데이터 전처리(결측치 임퓨테이션, 원-핫 인코딩 등) 및 알고리즘 탐색 노력을 최소화합니다.
- **높은 신뢰성과 결함 감내**: 실행 환경의 차이(특정 라이브러리 누락 등)에도 파이프라인이 즉시 크래시되지 않고, 최선의 유효 모델 세트를 활용하여 성공적인 완료를 지향합니다.
- **세련된 분석 결과 제공**: 의사결정권자 또는 연구원에게 분석 결과를 명확하게 인지시킬 수 있는 프리미엄 다크 테마 시각화 자료와 기계 가독성이 뛰어난 구조화된 JSON 리포트를 자동 발행합니다.

---

## 2. 시스템 요구사항 (System Requirements)

### 2.1 개발 및 실행 환경
- **언어**: Python 3.x
- **운영체제**: OS 독립적 (Linux, macOS, Windows 등 크로스 플랫폼 지원)

### 2.2 기술 스택 및 의존성 (Key Dependencies)
- **데이터 분석 및 프레임 처리**: `pandas` (>= 1.x), `numpy` (>= 1.x)
- **머신러닝 & 알고리즘**:
  - `scikit-learn`: 데이터 쪼개기(`train_test_split`), 평가 지표 계산, `RandomForestRegressor`, `MLPRegressor` 제공
  - `xgboost`: 트리 부스팅 기반 고성능 알고리즘 `XGBRegressor` 제공
  - `tabpfn`: 소/중형 정형 데이터 특화 사전 학습 정형 데이터 트랜스포머 모델 `TabPFNRegressor` 제공
- **시각화 및 파일 입출력**:
  - `matplotlib`, `seaborn`: 프리미엄 디자인 규격이 적용된 테마 차트 렌더링
  - `json`: 표준 구조화 리포트 저장을 위한 내장 객체
- **외부 데이터 소스 연동**:
  - `kaggle`: Kaggle API 연동 및 원격 데이터셋 파일 다운로드 지원
  - `urllib.request`: 내장 다운로더 모듈을 통한 직접 HTTP URL 파싱 지원

## 3. 기능적 요구사항 (Functional Requirements)

### 3.1 데이터 로드 및 수집 (DataLoader - Data Fetching & Loading)
시스템은 로컬에 이미 준비된 정형 데이터셋을 읽어올 뿐 아니라, 원격의 외부 플랫폼으로부터 데이터셋을 손쉽게 내려받아 가용화할 수 있어야 합니다.

* **REQ-DL-01: Kaggle 데이터 자동 연동 및 다운로드**
  - **설명**: Kaggle의 데이터셋 고유 식별자(예: `'sobhanmoosavi/us-accidents'`)를 제공받았을 때, 내장 Kaggle API 연동 모듈을 호출하여 데이터를 자동으로 다운로드하고 압축을 풀어 로컬 지정 디렉토리에 배치할 수 있어야 합니다.
  - **예외 처리**: 사용자 개발 환경에 `kaggle` 패키지가 설치되지 않았거나 API 인증키(`kaggle.json`)가 누락된 경우 예외를 조기에 포착하고, 관련 설치법 및 인증키 준비 경로(`~/.kaggle/`)를 가이드라인으로 출력해야 합니다.
* **REQ-DL-02: HTTP URL 데이터 자동 다운로드**
  - **설명**: UCI 머신러닝 리포지토리 등 외부 다이렉트 다운로드 주소(URL)를 입력받았을 때, 해당 주소로부터 원본 데이터를 지정된 파일명으로 직접 내려받아 로컬에 보관할 수 있어야 합니다.
  - **캐싱**: 이미 로컬 데이터 저장소에 동일한 파일이 존재하는 경우, 불필요한 네트워크 대역폭 낭비와 시간 지연을 방지하기 위해 다운로드 단계를 스킵(Skipping)해야 합니다.
* **REQ-DL-03: 다양한 데이터 포맷 파싱 및 대상 열 분리**
  - **설명**: 로컬의 `.csv`, `.tsv`, `.txt` (탭 구분자 포맷), `.parquet` 확장자를 자동으로 감지하여 적절한 판다스 판독 엔진을 통해 DataFrame 형태로 읽어 들일 수 있어야 합니다.
  - **검증**: 데이터프레임 내부에서 분석 대상 종속(타겟) 변수가 지정한 타겟명과 정확히 매칭하는지 유효성 검증을 거쳐야 하며, 해당 열이 부재한 경우 명확한 에러를 발생시키며 즉각 동작을 멈춰야 합니다.
  - **분리**: 검증 완료 후, 예측의 원인이 되는 특징(Feature)들로 이루어진 `X` 데이터와 예측 대상이 되는 타겟(Target) 컬럼 `y` 데이터로 분리하여 반환해야 합니다.

### 3.2 데이터 전처리 및 분할 (DataLoader - Preprocessing & Splitting)
모델 훈련에 완벽히 호환되는 최적의 데이터를 수립하기 위해 시스템은 정제 및 쪼개기 기법을 지원해야 합니다.

* **REQ-DL-04: 자동 결측치 보정 및 범주형 데이터 변환 (Preprocessing)**
  - **설명**: 수집된 원본 데이터의 특성을 스캔하여 모델 학습을 방해하는 결측치(Missing Values)와 텍스트 기반 범주형 값을 자동으로 보정해야 합니다.
  - **수치형 컬럼 정제**: 수치형 데이터셋 내부의 결측치는 데이터의 왜곡을 방지하기 위해 평균 대신 해당 열의 **중앙값(Median)**으로 자동 임퓨테이션(보정)합니다.
  - **범주형 컬럼 정제 & 인코딩**: 범주형 데이터 내의 결측치는 해당 열의 **최빈값(Mode)**으로 채우며, 이후 수학적 학습이 가능하도록 **더미 변수화(Dummy Encoding, 원-핫 인코딩)**를 자동으로 실행해야 합니다. 다중공선성(Multicollinearity) 문제를 방지하기 위해 첫 번째 범주를 제외하는 `drop_first=True` 전략을 고수합니다.
* **REQ-DL-05: 난수 고정을 포함한 데이터셋 분할 (Data Splitting)**
  - **설명**: 데이터를 학습(Train) 및 최종 성능 측정(Test) 셋으로 분할할 수 있어야 합니다.
  - **3-way 분할 지원**: 검증 데이터 비율(`val_size`)이 추가 파라미터로 제공될 경우, 전체 데이터를 학습 셋, 검증 셋, 테스트 셋의 3가지 묶음으로 정교하게 삼분할할 수 있어야 합니다.
  - **재현성**: 난수 시드(`random_state`)를 외부 매개변수로 받아 지정함으로써, 매 실행 때마다 완전히 무작위로 다르게 쪼개지는 현상을 막고 예측 결과의 동일성을 유지해야 합니다.

### 3.3 통일화된 모델 제어 및 모델 풀 구성 (ModelPool & ModelWrapper)
시스템은 여러 머신러닝 패키지의 서로 다른 API 호출 체계를 단일 인터페이스로 통일하고 동적으로 제어해야 합니다.

* **REQ-MP-01: 어댑터 패턴 기반 인터페이스 규격화 (ModelWrapper)**
  - **설명**: 다양한 외부 프레임워크 기반 모델(Scikit-Learn, XGBoost, TabPFN 등)의 제어 체계를 표준화하기 위해 `fit(X, y)` 및 `predict(X)`라는 공통 규격을 제공하는 **어댑터(Adapter) 패턴**을 내재화해야 합니다.
  - **타입 변환**: 모델별 요구 스펙에 대응하기 위해, Pandas DataFrame/Series로 인가된 학습 입력을 내부적으로 NumPy 다차원 배열로 안전하게 자동 캐스팅하여 처리해야 합니다.
* **REQ-MP-02: 기본 모델 풀 인스턴스화 (Default Models)**
  - **설명**: 머신러닝 알고리즘 군의 특성을 고려하여, 시스템 구동 시 회귀 문제에 최적화된 아래 4가지 유형의 대표 모델들을 적정 수준의 하이퍼파라미터 세팅으로 자동 빌드 및 관리 풀에 등재해야 합니다.
    1. **XGBoost (XGBRegressor)**: 트리 경사 부스팅 아키텍처 기반 알고리즘 (`n_estimators=100`, `learning_rate=0.1`, `n_jobs=-1` 설정)
    2. **MLP (MLPRegressor)**: 신경망 기반의 회귀 모델 (Hidden Layers: `(128, 64)`, Activation: `ReLU`, Solver: `Adam`, Max Iterations: `500` 설정)
    3. **TabPFN (TabPFNRegressor)**: 최근 각광받는 정형 데이터에 특화 사전 학습 트랜스포머 아키텍처 (소/중형 정형 데이터 분석 성능 극대화)
    4. **RandomForest (RandomForestRegressor)**: 배깅 앙상블 기법 기반의 신뢰도 높은 전통 트리 모델 (`n_estimators=100`, `n_jobs=-1` 설정)
* **REQ-MP-03: 패키지 부재 시의 생존력 보장 (Robust Import)**
  - **설명**: 사용자의 시스템 환경에 `xgboost`나 `tabpfn` 라이브러리가 미설치되었더라도, 메인 파이프라인 코드가 초기 부팅 과정에서 중단되어서는 안 됩니다. 해당 라이브러리가 없는 경우 런타임 경고(Warning) 로그를 출력하고, 가용한 나머지 모델들만으로 풀을 안전하게 구축해야 합니다.
* **REQ-MP-04: 커스텀 모델 동적 등록 (Dynamic Registration)**
  - **설명**: 시스템은 내장된 4대 모델 외에도 개발자가 자체적으로 구현했거나 튜닝한 임의의 외부 회귀 모델 인스턴스를 이름과 함께 동적으로 풀에 이식할 수 있는 확장 경로(`add_custom_model`)를 무조건 제공해야 합니다.
* **REQ-MP-05: 일괄 학습 및 예측 기능 (Batch fit & predict)**
  - **설명**: 관리 풀에 존재하는 모든 활성 모델을 일괄 훈련시키고(훈련 실패 시 해당 모델의 오류 정보를 로깅한 후 다음 모델 훈련을 지속하는 구조), 일괄 예측하여 결과를 각 모델명을 Key로 매핑한 딕셔너리로 즉각 반환할 수 있어야 합니다.

### 3.4 평가 지표 산출 및 리포팅 (Evaluation & Console Reports)
훈련을 종료한 모델들의 가치를 객관적으로 평가하고 판별할 수 있어야 합니다.

* **REQ-EV-01: 회귀 모델 전용 정량 메트릭 일괄 계산**
  - **설명**: 테스트 데이터 셋에 대해 모든 활성 모델별 예측값을 도출한 뒤, 실제값과 매칭하여 회귀 성능을 정밀 판정할 수 있는 3대 메트릭을 계산해야 합니다.
    - **RMSE (Root Mean Squared Error)**: 오차에 큰 패널티를 주는 평균 제곱근 오차
    - **MAE (Mean Absolute Error)**: 실제 오차의 절대값 평균 크기
    - **R² Score (R-squared / Coefficient of Determination)**: 회귀식의 설명력을 대변하는 결정 계수
* **REQ-EV-02: 콘솔 성능 비교 뷰 작성**
  - **설명**: 평가가 종료되는 즉시 표준 출력 터미널 창에 가로 구분선과 정렬된 서식을 적용하여 각 모델의 **RMSE, MAE, R² Score** 지표를 일목요연하게 비교한 텍스트 그리드 표(Grid Table)를 렌더링해야 합니다.

### 3.5 고급 다크 테마 시각화 및 실행 메타데이터 저장 (Visualizer)
분석 과정에서 도출되는 고도화된 통찰을 사람과 기계 모두 효율적으로 수용하도록 시각적 보존 장치를 제공합니다.

* **REQ-VI-01: 프리미엄 다크 테마 디자인 시스템 (Aesthetic Standards)**
  - **설명**: 출력되는 모든 플롯 파일에 직관성을 높이고 현대적인 세련됨을 주기 위해 일관된 **Premium Dark Theme**의 템플릿 스타일을 적용하여 빌드합니다.
  - **배경색(Canvas Background)**: 우아하고 깊은 차콜 블랙 슬레이트 테마 (`#0d1117`)
  - **도표 내부 배경색(Axes Face)**: 더 밝고 집중하기 쉬운 다크 그레이 컨테이너 (`#161b22`)
  - **그리드 라인(Grid Lines)**: 시선을 분산하지 않는 소프트 딤드 그레이 (`#30363d`)
  - **폰트/글씨 색상(Labels & Text)**: 고대비의 깔끔한 화이트 및 뮤티드 그레이 (`#c9d1d9`, `#8b949e`)
  - **하모니 파스텔 컬러 팔레트**: 블루(`#58a6ff`), 로즈코랄(`#ff7b72`), 민트그린(`#aff5b4`), 소프트퍼플(`#d2a8ff`), 머스타드옐로우(`#e3b341`)
* **REQ-VI-02: 실제값 vs 예측값 대칭 분석 차트 (Actual vs Predicted Scatter Plot)**
  - **설명**: 예측 오차의 일관된 쏠림을 시각화합니다. x축에 실제 정답 데이터, y축에 각 모델이 예측해낸 수치를 투명도가 가미된 파스텔 블루 점으로 플로팅하고, 완벽한 오차 없는 예측을 상징하는 **대각선 perfect fit line(y=x)**을 코랄 레드의 점선 형태로 명확하게 도식화해 저장해야 합니다.
* **REQ-VI-03: 잔차 분포 산포도 (Residual Scatter Plot)**
  - **설명**: 오차의 등분산성(Homoscedasticity) 확인 및 오차에 내재된 특정 편향 패턴을 찾기 위해, x축에 예측치, y축에 잔차(실제값 - 예측값)를 뿌려둔 산포도를 그리고 오차 0인 수평 기준선을 뚜렷하게 그려 넣어야 합니다.
* **REQ-VI-04: 일괄 성능 바 차트 (Model Comparison Horizontal Bar Chart)**
  - **설명**: 복수의 알고리즘군 중 어느 모델이 우세한지 판독할 수 있도록 모델명을 y축에, 해당 성능 수치를 x축에 표시한 수평 막대 차트를 생성해야 합니다.
  - **정렬 및 스펙 레이블**: 모델 성능 수준에 따라 순차적으로 자동 정렬되며, 막대의 오른쪽 끝 공간에 소수점 4자리까지의 정확한 지표 값을 표기하여 가독성을 높입니다.
* **REQ-VI-05: 이력 관리용 회차별 구조화 JSON 리포트 생성 (Structured Metadata Reporting)**
  - **설명**: 현재 AutoML 실행 회차 번호(Turn ID) 및 모델별 성능 평갓값(R², RMSE, MAE), 그리고 여러 모델 중 **가장 우수한 R² 점수를 획득한 최종 Champion Model명**을 추출하여 표준 JSON 리포트 파일(`turn_{turn}_report.json`)을 자동으로 기록 및 아카이빙해야 합니다.

### 3.6 CLI 인터페이스 및 엔드투엔드 파이프라인 조율 (main.py)
전체 모듈의 순차 조립과 사용자 명령 인수를 중앙 통제합니다.

* **REQ-PL-01: 다양한 파이프라인 제어 매개변수 인수화 (CLI Parser)**
  - **설명**: 명령 프롬프트 창에서 다음 인수를 유연하게 전달하여 시스템 작동 모드를 변경할 수 있게 합니다.
    - `--dataset-path`: 분석 타겟이 되는 로컬 CSV 파일의 저장 주소.
    - `--target`: 종속 대상 열의 레이블 명칭 (기본값: `"Target_Y"`).
    - `--test-size`: 평가에 격리 사용할 테스트 데이터 비율 (기본값: `0.2`).
    - `--turn`: 다수 실험 시 출력 산출물이 덮어써지지 않도록 넘버링하는 실행 회차 인덱스 (기본값: `1`).
    - `--kaggle-dataset`: 원격 Kaggle 다운로드 트리거용 고유 식별 명칭.
    - `--url`: 외부 UCI 등 HTTP 원격 데이터 다이렉트 다운로드 주소.
* **REQ-PL-02: 다운로드 실패 또는 미지정 시 모의 회귀 데이터셋 생성 (Zero-Failure Fallback)**
  - **설명**: 로컬 CSV 경로를 적지 않았거나, Kaggle/URL 인자가 없고, 원격 내려받기에 완전히 실패한 경우라 하더라도 프로그램이 즉각 에러로 정지하지 않고, **비선형 합성 회귀 수식 관계를 내포한 고품질 모의 Tabular 데이터셋(`Feature_X1`, `Feature_X2`, `Category_X3`, `Target_Y` 구성)**을 즉시 생성하여 `"data/synthetic_regression.csv"` 경로에 자가 탑재 후 파이프라인을 온전히 구동 완료해야 합니다.

## 4. 비기능적 요구사항 (Non-Functional Requirements)

### 4.1 사용성 및 접근성 (Usability & Out-of-the-box Execution)
- 사용자는 복잡한 환경 설정 파일이나 복잡한 파이썬 스크립트 수정 필요 없이, `python automl_framework/main.py` 명령어 하나만으로 곧바로 프레임워크 전체 프로세스(데이터 생성, 전처리, 다중 알고리즘 병렬 훈련, 성능 평가 테이블 콘솔 뷰어 출력, 시각화 파일 출력, JSON 리포팅)를 즉시 목격할 수 있어야 합니다.

### 4.2 강인성 및 결함 감내 (Robustness & Exception Tolerant)
- `tabpfn`, `xgboost` 등 하드웨어 가속이나 플랫폼 환경에 기인하여 설치 장벽이 높거나 버그가 날 수 있는 패키지 수집 및 훈련 프로세스 전체에 대해 철저한 방어형 예외 처리(Try-Catch Wrapper)를 수행하여 일부 결함이 시스템 전반의 붕괴로 번지는 것을 전적으로 예방합니다.

### 4.3 출력물 체계화 및 분리 아카이빙 (Artifact Segregation & Multi-Turn Support)
- 원본 및 모의 데이터 파일은 오로지 `"data/"` 디렉토리에만 보관하고, 차트 분석 이미지(.png) 및 실행 메타데이터 JSON 파일은 오로지 `"outputs/"` 디렉토리에만 정돈하여 저장합니다.
- 실험 회차 지표 `--turn` 값을 기준으로 결과 파일명에 번호를 고유하게 부여하여, 수많은 반복 테스트 시에도 이전 결과가 덮어씌워지며 망실되는 것을 철저히 차단합니다.

## 5. 데이터 흐름 및 실행 아키텍처 (Data Flow Diagram)

시스템의 파이프라인 실행에 따른 모듈 간 데이터 교환 흐름은 아래와 같습니다.

```text
[1. CLI Entry]  ─────> [2. Data Ingestion] ─────> [3. Data Preprocessor] ─────> [4. Data Splitting]
   (main.py)           (DataLoader.load)         (DataLoader.preprocess)       (DataLoader.split)
                                                                                       │
                                                                                       ▼
[8. Champion Select] <── [7. Premium Charts] <─── [6. Evaluation Metrics] <─── [5. Model Pool Fit]
   (outputs/JSON)         (Visualizer)             (ModelPool.evaluate)         (ModelPool.fit)
```

1. **CLI Entry**: 사용자가 터미널창에 매개변수를 기입하여 실행을 명령합니다.
2. **Data Ingestion**: Kaggle API, UCI HTTP, 로컬 CSV 판독 또는 모의 자가 생성 루틴을 돌려 원본 데이터프레임을 생성하고 타겟 벡터를 분리합니다.
3. **Data Preprocessor**: 누락 값 검측 후 수치열은 중앙값, 범주열은 최빈값 임퓨팅 및 원-핫 인코딩(Dummy 변수 변환)을 거쳐 정규 매트릭스로 조율합니다.
4. **Data Splitting**: 재현용 난수 시드를 걸어 학습 및 평가용 테스트 데이터로 조각냅니다.
5. **Model Pool Fit**: `ModelWrapper`로 감싸 인터페이스가 통일된 가용 모델(XGBoost, MLP, TabPFN, RandomForest)에 데이터를 투입하여 일괄 학습시킵니다.
6. **Evaluation**: 계산된 예측값을 오리지널 정답과 매칭해 RMSE, MAE, R² 메트릭으로 요약하여 콘솔 표로 표시합니다.
7. **Premium Charts**: 고급 다크 슬레이트 테마 레이아웃 위에 실제값 대 예측값 분포, 잔차 오차 분포, 모델 성능 순위 수평 바 플롯을 그려 고화질 PNG 이미지로 기록합니다.
8. **Champion Select**: 훈련된 모든 알고리즘군 중 R² 성능이 가장 뛰어난 1위 최적의 모델(Best Model) 명칭을 검출하여 JSON 아카이브 리포트 파일 저장을 끝으로 모든 시퀀스를 무사히 완료합니다.
